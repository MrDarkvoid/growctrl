# Lizenz: GROWCTRL Source-Available (GC-SAL 1.0)

GROWCTRL ist **source-available**, nicht Open Source: ansehen und privat
nutzen ja - kommerzielle Nutzung, Re-Hosting und veroeffentlichte
Modifikationen nur mit schriftlicher Zustimmung von **MrDarkvoid**.
Der vollstaendige Text steht in [`LICENSE`](../LICENSE).

## So stellst du die Lizenz auf GitHub um
1. Repository oeffnen -> Datei `LICENSE` -> Stift-Symbol (Edit) ->
   alten MIT-Text durch den GC-SAL-Text ersetzen -> **Commit changes**.
2. Pruefen, dass keine zweite Lizenzdatei existiert (`LICENSE.md`,
   `COPYING`) - sonst loeschen.
3. Repo-Startseite -> Zahnrad neben "About" -> Beschreibung ergaenzen,
   z.B. "Source-available - keine kommerzielle Nutzung".
4. Optional ein Badge in die README (ist hier schon eingebaut).

## Ehrliche Hinweise
- **GitHub zeigt im Repo-Header nur Standard-Lizenzen an** (MIT, GPL, ...).
  Bei einer eigenen Lizenz steht dort "View license" - das ist normal.
- **Forken innerhalb von GitHub laesst sich nicht verbieten**: Die GitHub
  Terms of Service raeumen jedem Nutzer das Recht ein, oeffentliche Repos
  auf der Plattform anzusehen und zu forken. Die Lizenz greift fuer alles
  DARUEBER hinaus: Nutzung, Verbreitung, eigene Releases, Modifikationen.
  Wer das Plattform-Forken komplett verhindern will, muss das Repo privat
  stellen.
- **HACS**: Custom Repositories funktionieren weiter; fuer eine Aufnahme
  in die HACS-Default-Liste kann eine OSI-Lizenz verlangt werden.
- Bestehende Klone unter der frueheren MIT-Lizenz bleiben unter MIT -
  ein Lizenzwechsel wirkt nur fuer neue Staende ab dem Wechsel-Commit.
